title: Data Model
description: Schema overview for all tables in the Atom database.
---

All primary keys are UUIDs generated by Postgres (`gen_random_uuid()` via pgcrypto). All timestamps are `TIMESTAMPTZ` in UTC.

## entities

Every principal in the system.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `kind` | text | `human \| device \| service \| workload \| application` |
| `name` | text | Unique per tenant |
| `tenant_id` | uuid nullable | Scopes the entity to a tenant |
| `status` | text | `active \| inactive \| suspended` |
| `attributes` | jsonb | Arbitrary key-value metadata, usable in ABAC conditions |
| `created_at` | timestamptz | |
| `updated_at` | timestamptz | |

Indexes: `kind`, `tenant_id`, `name`, `(name, tenant_id)` unique, GIN on `attributes`.

## credentials

Authentication credentials attached to an entity.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | Embedded in API keys |
| `entity_id` | uuid FK → entities | Cascade delete |
| `kind` | text | `password \| api_key \| certificate` |
| `identifier` | text nullable | Username for passwords; key prefix for API keys |
| `secret_hash` | text nullable | argon2 hash |
| `metadata` | jsonb | e.g. `{"description": "device-01 production key"}` |
| `status` | text | `active \| revoked` |
| `expires_at` | timestamptz nullable | |
| `created_at` | timestamptz | |

## sessions

Tracks active authentication sessions.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | Referenced by JWT `sid` claim |
| `entity_id` | uuid FK → entities | |
| `expires_at` | timestamptz | |
| `revoked_at` | timestamptz nullable | Set by logout |
| `created_at` | timestamptz | |

## groups / group_members

Named collections of entities.

**groups**: `id`, `name` (unique per tenant), `tenant_id`, `description`, `created_at`, `updated_at`.

**group_members**: `(group_id, entity_id)` composite PK.

## ownerships

Parent–child relationships between entities.

| Column | Type |
|---|---|
| `owner_id` | uuid FK → entities |
| `owned_id` | uuid FK → entities |
| `relation` | text (default `"owner"`) |
| `created_at` | timestamptz |

`(owner_id, owned_id)` is the composite PK.

## resources

Anything that can be protected by authorization.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `kind` | text | e.g. `channel`, `device`, `workspace` — free-form |
| `name` | text nullable | |
| `tenant_id` | uuid nullable | |
| `owner_id` | uuid nullable FK → entities | |
| `attributes` | jsonb | Usable in ABAC conditions |
| `created_at` / `updated_at` | timestamptz | |

## capabilities

Atomic permission names.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `name` | text | e.g. `publish` |
| `resource_kind` | text nullable | `NULL` means applies to all resource kinds |
| `description` | text nullable | |

Unique constraint on `(name, resource_kind)`.

**Seeded defaults** (resource_kind = NULL): `read`, `write`, `delete`, `publish`, `subscribe`, `execute`, `manage`.

## roles / role_capabilities

Named bundles of capabilities.

**roles**: `id`, `name` (unique per tenant), `tenant_id`, `description`, `created_at`.

**role_capabilities**: `(role_id, capability_id)` composite PK.

## policy_bindings

The central authorization primitive. See [Policy Bindings](/docs/access-control/policies) for full reference.

| Column | Type |
|---|---|
| `id` | uuid PK |
| `subject_kind` | text — `entity \| group` |
| `subject_id` | uuid |
| `grant_kind` | text — `capability \| role` |
| `grant_id` | uuid |
| `scope_kind` | text — `all \| resource_kind \| resource` |
| `scope_ref` | text nullable |
| `effect` | text — `allow \| deny` |
| `conditions` | jsonb — ABAC conditions |
| `created_at` | timestamptz |

## signing_keys

ES256 key pairs for JWT signing.

| Column | Type | Notes |
|---|---|---|
| `kid` | text PK | UUID string, embedded in JWT `kid` header |
| `algorithm` | text | `ES256` |
| `public_key` | text | SubjectPublicKeyInfo PEM |
| `private_key` | text | PKCS8 PEM |
| `status` | text | `primary \| standby \| retired` |
| `created_at` | timestamptz | |

At most one `primary` and one `standby` key exist at any time.

## audit_logs

Immutable event record.

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `entity_id` | uuid nullable FK → entities | |
| `event` | text | e.g. `auth.login`, `authz.check`, `credential.create` |
| `outcome` | text | `allow \| deny \| error` |
| `details` | jsonb | Event-specific context |
| `created_at` | timestamptz | |
