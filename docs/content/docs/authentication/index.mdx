title: Authentication
description: Token types, credential management, and how authentication works in Atom.
---

Atom supports two token types, both passed as a Bearer token in the `Authorization` header:

```
Authorization: Bearer <token>
```

| Token type | Format | Use case |
|---|---|---|
| JWT | `eyJ...` | Short-lived (default 1h), returned by `/auth/login` |
| API key | `atom_...` | Long-lived, created per entity, optional expiry |

## Public endpoints

The following endpoints do not require authentication:

```
GET  /health
POST /auth/login
GET  /.well-known/jwks.json
```

All other endpoints require a valid Bearer token.
