## Prerequisites

- Docker and Docker Compose
- Rust toolchain
- `jq` for command examples

## Steps

<Steps>
<Step>

### Start Postgres

```bash
docker compose up postgres -d
```

</Step>
<Step>

### Configure Atom

```bash
cp .env.example .env
```

Set at least:

| Variable | Description |
|---|---|
| `DATABASE_URL` | Postgres connection string. |
| `JWT_SECRET` | 32+ character secret for JWT compatibility paths. |
| `ADMIN_SECRET` | Initial password for `atom-admin` on first boot. |

If certificates are enabled, mount issuer CA files as described in [Certificates](/authentication/certificates).

</Step>
<Step>

### Run Atom

```bash
cargo run
```

Atom applies migrations automatically.

</Step>
<Step>

### Log In

```bash
TOKEN=$(curl -s -X POST http://localhost:8080/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"identifier": "atom-admin", "secret": "your-admin-secret"}' \
  | jq -r .token)
```

</Step>
<Step>

### Create A Tenant, Device, And Channel

Use GraphQL at `POST /graphql`.

```bash
curl -s http://localhost:8080/graphql \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "mutation { createTenant(input: { name: \"factory-a\", route: \"factory-a\" }) { id name route } }"
  }'
```

Create an entity for a device:

```graphql
mutation {
  createEntity(input: {
    kind: device
    name: "meter-001"
  }) {
    id
    name
    kind
  }
}
```

Create a channel resource:

```graphql
mutation {
  createResource(input: {
    kind: "channel"
    name: "telemetry"
  }) {
    id
    kind
    name
  }
}
```

</Step>
<Step>

### Give Publish Access

Find the seeded `publish` action:

```graphql
{
  actions(objectKind: "resource", objectType: "resource:channel") {
    items {
      id
      name
    }
  }
}
```

Create a permission block for the channel:

```graphql
mutation {
  createPermissionBlock(input: {
    scopeMode: "object"
    objectId: "channel-resource-id"
    effect: allow
    actionIds: ["publish-action-id"]
  }) {
    id
    effect
    actions { name }
  }
}
```

Create a role and attach the permission block:

```graphql
mutation {
  createRole(input: {
    name: "channel-publisher"
    description: "Can publish telemetry"
  }) {
    id
    name
  }
}
```

```graphql
mutation {
  replaceRolePermissionBlocks(
    roleId: "role-id"
    permissionBlockIds: ["permission-block-id"]
  )
}
```

Assign the role to the device:

```graphql
mutation {
  createRoleAssignment(input: {
    subjectKind: entity
    subjectId: "device-entity-id"
    roleId: "role-id"
  }) {
    id
    subjectId
    roleId
  }
}
```

</Step>
<Step>

### Check Authorization

```graphql
mutation {
  authzCheck(input: {
    subjectId: "device-entity-id"
    action: "publish"
    resourceId: "channel-resource-id"
  }) {
    allowed
    reason
  }
}
```

Expected result:

```json
{
  "allowed": true,
  "reason": "allowed"
}
```

</Step>
</Steps>

## What Just Happened

1. You created an entity for a device.
2. You created a resource for a channel.
3. You created a permission block that allows `publish` on that channel.
4. You put the permission block inside a role.
5. You assigned the role to the device.
6. Atom returned allow when asked whether the device can publish.

See [Atom In Simple Words](/simple-words) for the layman explanation, and [Access Control](/access-control) for the full model.
